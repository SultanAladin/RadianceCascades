## Plugin pattern (load-bearing principle, reference)

Every swappable algorithm family has an abstract interface in `Source/<Family>/I<Family>Algorithm.h`:

* `IShadowAlgorithm` ‚Äî PCF / PCSS / VSM / SDFCone implementors
* `IGIAlgorithm`     ‚Äî SDFGI (only v1 implementor; voxel-cone / radiance-cascades drop in later)

`Renderer::Impl` holds `std::unique_ptr<IXxx>` per family + an enum + a setter. Switching algos = `Terminate()` old ‚Üí `make_unique<NewImpl>()` ‚Üí `Initialize()` new ‚Üí rebuild lighting pipeline with new specialisation constant.

**Descriptor contract** (the load-bearing detail): the lighting compute shader reserves:
* `set=2, binding=0` for the shadow algo's output image (sampler2D / sampler2DShadow / R8 / RG32F depending on algo, with a spec-constant variant switch)
* `set=2, binding=1` for a 256-byte algo-private UBO
* `set=3, binding=0` for the GI algo's half-res output (RGBA16F)

Lighting never branches on `which algo`; the pipeline is recompiled on swap.

## Original 14b plan (preserved for reference)

**Goal:** with 14a groundwork (settings, plugin, ImGui panel, hot-swap) + Taichi validation in the bag, build the GPU side of RC √ó Hash. Concretely:

1. **Hash + payload SSBOs.** Two device-local buffers sized 2^20:

2. **`Shaders/rc_insert.comp`** ‚Äî dispatch one workgroup per visible pixel of the GBuffer (downsampled 4√ó, ‚âà 80k threads at 1280√ó720). Each thread:
   - Reads world-pos from `InvViewProj * (uv, depth)` (the lighting compose already does this ‚Äî copy the helper into `Shaders/include/gbuffer_decode.glsl` if you want one source of truth).
   - Computes cascade level from `length(eye - worldPos)` (matches the lighting cascade-selection logic).

   - **Secondary insertion pass.** After the visible-pixel pass, dispatch a second workgroup grid that walks the **camera frustum extruded by N cells** (try N=2 cells at cascade 0, growing with cascade) and inserts probes for off-screen surfaces. This is the fix for the **11.5% screen-only coverage** that Taichi C3 measured. Implementation: dispatch a 3D grid sized to the extended frustum; each thread votes whether its cell is "likely visible soon" and inserts. Cheap because we never trace SDF ‚Äî just hash + memset SH=0.
3. **`Shaders/rc_relight.comp`** ‚Äî one workgroup per occupied cell (drive from a compacted occupied-cells SSBO produced by a parallel scan, or just dispatch `2^20 / 64` workgroups and `early-out` on empty slots ‚Äî at 1.32% load the wasted work is negligible).
   - For each cell, trace `R0=32` rays into the SDF using the same `improved Quilez` primitive from `lighting_sdfcone.comp` (now's the time to extract `Shaders/include/sdf_trace.glsl`).
   - At ray endpoint: if hit, sample direct lighting (sun ¬∑ NdotL ¬∑ visibility) at the hit point; if miss, look up the next-cascade SH probe at the endpoint (the **radiance-interval extension** validated by Taichi C4).
   - Accumulate into SH-L1 with the standard `Y_00 = 0.282`, `Y_1m1/Y_10/Y_11 = ¬±0.488 ¬∑ dir.{x,y,z}` basis.
   - Write back to `ProbePayload`.
4. **`GIDebugView` viz wired in.** Each mode is a single-purpose render branch in `gbuffer_preview.frag` (or a new `gi_debug.comp`):
   - `HashCells` ‚Äî colour by `hash_word(cellKey) % 256` so eyeball-distinct cells.
   - `Cascades` ‚Äî colour by `cascadeLevel` (R/G/B/A bands).
   - `ProbeRays` ‚Äî for one probe under the cursor, raster 32 line segments showing the trace directions (uses the pick-readback identity buffer to grab the cursor cell).
   - `Irradiance` ‚Äî SH-L1 evaluated at the surface, no albedo modulation. This is the smoking-gun test: sun rotates ‚Üí SH cells visibly rebake.
